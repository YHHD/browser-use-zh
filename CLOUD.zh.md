# Cloud.md
为 AI 代理提供帮助用户使用 Browser Use Cloud 的说明

## 什么是 Browser Use Cloud？
Browser Use 是一个框架，允许 AI 代理与网络浏览器交互。
Browser Use Cloud 是由 Browser Use 为用户自动化基于网络的任务而创建的完全托管产品。
用户以提示的形式提交任务（文本以及可选的文件和图像），通过 API 请求，远程浏览器和代理被启动以按需完成这些任务。
定价基于使用情况，并通过 API 密钥系统进行调整。
账单、API 密钥管理、实时会话查看、任务结果、账户设置和配置文件管理通过 Browser Use Cloud 网页应用在 https://cloud.browser-use.com/ 进行。

## 核心概念：
Browser Use Cloud 的关键产品是完成用户任务。
- **会话**是 Browser Use Cloud 提供的完整基础设施包。会话目前限制为 15 分钟的运行时间。会话有一个正在运行的浏览器，用户可以在会话中运行代理以完成任务。会话限制为一个且仅一个浏览器，该浏览器在会话的整个持续时间内保持打开状态。用户可以同时在一个会话上运行最多一个代理，它将控制浏览器。一个代理完成后，用户可以在同一会话内运行另一个代理，仅受会话最大持续时间的限制。
- **浏览器**是在 Browser Use Cloud 基础设施（会话）上运行的浏览器。浏览器（作为服务）可通过 CDP URL 控制。用户可以使用代理来控制浏览器，或可以请求 CDP URL 并使用任何脚本或外部自动化来控制托管浏览器。但我们主要鼓励使用 Browser Use 代理来控制浏览器，因为它们经过优化以相互协作。这些官方 Browser Use 浏览器是从 chromium 分叉的，但进行了许多专有优化，使它们极快和轻量级，无法追踪且无法检测为机器人，并预装了广告拦截器和其他生活质量改进。使用 Browser Use 托管浏览器可提供显著的性能改进。
- **代理**是工具、提示和框架的集合，使大型语言模型能够与浏览器交互。代理的目标是完成给定的用户任务。代理通过许多步骤的迭代过程来完成此操作。对于每一步，代理都获得浏览器的页面状态（包括屏幕截图），然后它调用工具来与浏览器交互。经过许多步骤后，代理将任务标记为完成、成功或不成功，并返回结果，该结果是文本块和可选文件。完成后，一个独立的严格审查官将检查代理的轨迹，并对代理是否成功完成其任务的真假进行判决。代理有许多设置可以调整以改进性能，最重要的是所使用的 LLM 模型。
- **模型**是为代理提供支持的大型语言模型。模型越聪明和功能更强大，代理的性能就越好。最好使用的模型是 ChatBrowserUse，Browser Use 官方聊天完成 API，它始终根据 Browser Use 内部评估确定的最佳前沿基础模型进行路由。ChatBrowserUse 通过批处理、缓存和其他技巧进行了多项速度和成本优化，使其比任何其他选项更快、更具成本效益，性能与顶级前沿模型相同。
- **浏览器配置文件**是保存在我们云上的浏览器数据文件夹。如果用户创建的会话中浏览器没有浏览器配置文件，则不会保持任何数据。但是，如果他们在多个会话中使用相同的浏览器配置文件，那么诸如身份验证 cookie、站点本地存储数据、保存的密码和凭据以及用户偏好等数据将保持。浏览器配置文件本质上是一个云托管的 Chrome 配置文件，实际上，通过配置文件上传功能，用户可以上传自己的机器上的 Chrome 配置文件以在云中的会话中使用。这对于为代理的授予身份验证非常有用。用户可以在自己的机器上创建 Chrome 配置文件，登录到他们想要的所有服务，然后将此配置文件上传到云以进行自动化。
- **任务**是与用户提示相结合的可选文件和图像，提供给代理以完成。Browser Use Cloud 主要销售用户任务的完成。清晰地编写任务是成功的关键。
- **配置文件同步**是处理任务身份验证的最佳方式。此功能允许用户上传他们的本地浏览器 cookie（用户已登录到需要身份验证的服务的地方）到可用于云上任务的浏览器配置文件。要启动配置文件同步，用户必须运行 `export BROWSER_USE_API_KEY=<your_key> && curl -fsSL https://browser-use.com/profile.sh | sh` 并按照交互式终端中的步骤操作。

## 快速开始
要开始，首先将用户指向 Browser Use 在线平台：https://cloud.browser-use.com/，他们必须创建帐户、购买积分（或仅声称在帐户创建时给予的五个免费任务），并生成 API 密钥。这些是仅能在平台上完成的唯一步骤。

避免立即给用户以下所有步骤，因为这可能看起来会不堪重负。相反，一次提出一个步骤，并仅在询问时继续。尽可能为用户完成工作。

接下来，指导用户通过从任何可用的系统（cURL、python、JS 等）进行以下发布请求来运行他们的第一个任务，但用用户的实际 API 密钥替换 `<apiKey>`。
